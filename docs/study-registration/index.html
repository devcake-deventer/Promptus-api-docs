Study registration

Edit study registration - Administrator only

When a user wants to take part in a study the user can register for it. In those situations a study registration is created and saved for that specific study. For a number of reasons it might be necessary to change or update this registration.

The mutation that can be used for these types of situations is the: updateStudyRegistration. Do take note that only administrators can use this mutation to update a study registration.

mutation {
  updateStudyRegistration(id: 2, graduated: false) {
    id
    graduated
  }
}

Required parameters

This mutation only requires a single parameter: the id of the registration that should be updated. If the id is not provided or a study registration with the id could not be found an error will be thrown.

Optional paramters

In the table below you'll find a comprehensive list of all the optional parameters supported for this mutation. Providing any other parameter will result in an exception being thrown.

FieldTypeDescription
pricePriceThe price of the course registration
freeBooleanIndicates if this study registration is free
scale_amountIntAmount of concurrent course registration made when registering for this study
registration_numberStringIf you want to specify a custom registration number, this option can be used
registration_dateDateTimeThe date the registration was done
confirmation_dateDateTimeThe date the registration was confirmed
cancellation_dateDateTimeThe date the registration was cancelled
cancellation_dateDateTimeThe date the registration was cancelled
particularitiesStringIf there are any particularities regarding this registration, you can specify them here
graduatedBooleanIndicates if the the study that the user has enrolled for with this registration, has been succesfully completed
payment_attnStringName to use for correspondence
btwTypeIDUnique identifier that associates an existing btw type with this study registration
debtorCompanyIDUnique identifier that associates an existing company with this study registration
debtorTypeIDUnique identifier that associates an existing debtor type with this study registration (1)
priceTypeIDUnique identifier that associates an existing price type with this study registration (1)
debtorThirdPartyIDUnique identifier that associates an existing company with this study registration
paymentSchemeIDUnique identifier that associates an existing payment scheme with this study registration (1)
paymentMethodIDUnique identifier that associates an existing payment method with this study registration (1)
correspondenceCompanyIDUnique identifier that associates an existing company with this study registration
correspondenceUserIDUnique identifier that associates an existing user with this study registration for correspondence

(1) Refer to their respective API tables to see available options

Describing data to receive

Between the {} of the mutation you can specify the information you want to receive once the mutation is successfull. When we run the mutation at the top of this chapter the result returned should look like the following.

{
  "data": {
    "updateStudyRegistration": {
      "id": "1",
      "graduated": false
    }
  }
}

Edit course registration

When a user wants to take part in a course the user can register for it. In those situations a course registration is created and saved for that specific course. For a number of reasons it might be necessary to change or update this registration.

For these kind of situtations we offer two different mutations. The first can be used for a authenticated user to update information about the users own registration(s). The second is a mutation available for administrators to update another user's registration.

Edit a course registration - Administrator only

Users with administrative rights are able to edit other user's course registrations. This enables them to offer support whenever this seems necessary through the updateCourseRegistration mutation.

mutation {
  updateCourseRegistration(id: 2, comment: "example comment") {
    id
    comment
  }
}

Required parameters

This mutation only requires a single parameter: the id of the registration that should be updated. If the id is not provided or a course registration with the id could not be found an error will be thrown.

Optional parameters

In the table below you'll find a comprehensive list of all the optional parameters supported by this mutation. Providing any other parameter will result in an exception being thrown.

FieldTypeDescription
pricePriceThe price of the course registration
confirmation_dateDateTimeThe date the registration was confirmed
cancellation_dateDateTimeThe date the registration was cancelled
placement_dateDateTimeThe date the user corresponding to this registration was placed in a class or group
material_sent_dateDateTimeThe date the course materials were sent
certificate_dateDateThe date provided on the certificate
commentStringA comment regarding this course registration
queuedBooleanIndicating if the course registration is queued. Can be used when max participants is exceeded
discountFloatIndicates the percentage of discount applied to this registration
payment_attnStringName to use for correspondence
purchase_numberStringNumber that identifies the purchase
examIDUnique identifier that associates an existing exam (result) with this course registration
priceTypeIDUnique identifier that associates an existing price type with this course registration (1)
debtorTypeIDUnique identifier that associates an existing debtor type with this course registration (1)
billingCompanyIDUnique identifier that associates an existing company with this course registration
debtorThirdPartyIDUnique identifier that associates an existing company with this course registration
paymentSchemeIDUnique identifier that associates an existing payment scheme with this course registration (1)
paymentMethodIDUnique identifier that associates an existing payment method with this course registration (1)
correspondenceCompanyIDUnique identifier that associates an existing company with this course registration
correspondenceUserIDUnique identifier that associates an existing user with this course registration for correspondence

(1) Refer to their respective API tables to see available options

Describing data to receive

Between the {} of the mutation you can specify the information that is sent back to the client once the mutation is successfull. When we run the mutation at the top of this chapter the result returned should look like the following example.

{
  "data": {
    "updateCourseRegistration": {
      "id": "2",
      "comment": "example comment"
    }
  }
}
